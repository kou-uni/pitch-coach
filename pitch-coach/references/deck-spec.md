# デッキ生成仕様（Phase 7用）

pptxgenjs（Node.js）で生成する。ゼロから書かず、`assets/deck_template.js` を作業ディレクトリにコピーしてテキストを差し替えるのが基本。レイアウト座標・ヘルパー関数（header/pill/card）は検証済みなので変えない。

準備: `npm install pptxgenjs`（グローバル導入済みなら `NODE_PATH` を通す）。

## デザイントークン

- レイアウト: LAYOUT_16x9（10" × 5.625"）
- 色: NAVY=2F3C7E（ダーク面背景）、NAVY_DK=232C5C（ダーク面カード）、CORAL=F96167（アクセント）、GOLD=F9E795（時間ピル）、INK=1E2430（本文）、GRAY=6B7280、TINT=F4F5FA（ライト面カード）、TINT_CORAL=FEF0F0
- フォント: "Yu Gothic"。タイトル24pt（S1は32pt・2行可、S9は26pt）、本文12〜13pt、脚注10.5pt
- S1とS9はNAVY背景（ダーク）、S2〜S8は白背景のサンドイッチ構造

## 各スライドの型

- **S2〜S8共通ヘッダー**: コーラル円の番号 → NAVYピルにVDS要素名（課題設定/市場性/顧客価値/競争優位性/実現可能性/収益性）→ GOLDピルに秒数 → 下に結論文タイトル（24pt bold）
- **S1**: NAVY背景。価値提案文32pt中央（「【誰】が、できなかった【○○】を、できるようにする」の型）、プロダクト名20ptコーラル、下端に氏名・所属
- **S2**: 左に時系列エピソードカード（箇条書き4点: いつ/何が/どう感じた/既存手段がダメだった理由）、右に感情フックカード（逃した価値などを34ptで大きく）
- **S3**: 一般化の一文 → 数字カード3枚（数字36pt・ラベル12pt・出典8.5ptを縦積み）
- **S4**: 左に3ステップ（番号円＋縦線接続）、右にデモ録画プレースホルダ（NAVY_DKカード＋再生ボタン風のコーラル円＋「デモ録画（○秒）」）
- **S5**: 「今までの世界」（TINT・グレー文字18pt）→ コーラル矢印 →「これからの世界」（TINT_CORAL・コーラル文字18pt）。下端にβ検証実績の1行（コーラル・イタリック）
- **S6**: 「N1当事者性」×「検証速度」の2カード（間にコーラルの×）。下端に既存サービスへの構造的差別化1行
- **S7**: 左カードにエージェント構成フロー図（NAVY角丸ボックス＋矢印＋各役割の8.5ptキャプション、下に品質担保の1行）、右に開発期間/イテレーション/自分で書いたコード比率の3カード（TINT_CORAL）
- **S8**: 中央に「単価 × 件数 = 年商」の1式カード（22pt）＋根拠1行
- **S9**: NAVY背景。ビジョン結論文26pt → 3カードのロードマップ（次の○週間/次の1年/その先）。3枚目は固定文言「新しい『できなかった』を見つけて、また作る」。下端に連絡先

## スピーカーノート

各スライドに `slide.addNotes()` で「【話す（○秒）】」から始まる原稿を入れる。受講生の一人称・対話で使った言葉をなるべく残す。原稿の後に「※」で発表時の注意（例: 感情の起伏はこの1枚だけ／機能一覧は語らない）を1行添える。

## pptxgenjsの落とし穴（違反するとファイル破損または崩れ）

1. 色に `#` を付けない（`"FF0000"` が正。`"#FF0000"` は破損）
2. 8桁hexで透明度を書かない。shadowの透明度は `opacity` プロパティ（0.0〜1.0）で
3. 箇条書きは `bullet: true`。「・」「•」を文字で書かない（二重弾丸になる）
4. 複数行テキストは各runに `breakLine: true`
5. オプションオブジェクトを複数の addShape/addText で使い回さない（pptxgenjsが破壊的に書き換えるため2回目が壊れる。`() => ({...})` で毎回生成する）
6. 装飾の色バー・エッジストライプを置かない。カードの区別は塗り＋シャドウで
7. `lineSpacing` を箇条書きに使わない。`paraSpaceAfter` を使う

## 生成後の必須チェック

1. 日本語タイトルの折り返し: 24ptで1行に入るのは約27文字。超えるとカードに重なるので、**まずタイトルを短くする**（フォント縮小は最終手段）
2. 数字カード内の大数字が折り返していないか（36ptで約7文字まで）
3. LibreOfficeがあれば PDF化 → 画像化して目視確認:
   ```bash
   soffice --headless --convert-to pdf output.pptx
   pdftoppm -jpeg -r 150 output.pdf slide
   ```
   なければ受講生に開いてもらい、はみ出し箇所を聞いて修正する
4. pptxgenjsの書き出しは無圧縮ZIPになるので、Pythonが使えるならzipfileで再圧縮するとファイルサイズが数分の一になる（必須ではない）
